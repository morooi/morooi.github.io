Trojan 与 v2ray 通过 Caddy 共存及在 macOS 中的使用

2020 年 8 月 23 日更新

新方法，利用 Nginx 在第三层进行流量分流。

具体参考：

https://www.chengxiaobai.cn/record/trojan-shared-443-port-scheme.html

https://www.v2rayssr.com/trojan-bt.html

很早就听说 Trojan 速度比 v2ray 快，今天突然看到 Quantumult X 更新到 1.0.7 版本，并且加入了 Trojan 的支持，于是想搭个试试，并且保证不影响 v2ray 的使用，但是不知道会不会影响使用 v2ray 的性能

Trojan 项目地址：https://github.com/trojan-gfw/trojan

过程：

  • 安装 BBR Plus
  • 使用 233boy 一键脚本 搭建 v2ray，并使用 ws + tls 方式
  • 使用 Trojan 官方脚本 安装 Trojan
  • 配置 Caddy
  • 配置 macOS
  • 配置 iOS

本文不解释原理，只是记录下搭建和配置的过程，供自己以后备用

修改 Caddy 配置

修改 /etc/caddy/Caddyfile，使 Caddy 只监听 80 端口

Trojan 服务监听 443 端口，将与 Trojan 无关的流量转移到 80 端口，Caddy 监听到并转发给 v2ray 处理，实现共存
没有实测，但多了一层转发，理论上会影响 v2ray 的性能
