使用条件

本文写于 2022.12.3，后续软件可能有更新

  • macOS 平台暂无软件支持，需安装 Xray 二进制程序手动运行，然后通过 v2rayU、clashX、clashX Pro 等代理软件连接
  • iOS /iPadOS 平台暂无软件支持，可使用具有公网 IP 的服务器搭建任意代理服务（如 Shadowsocks）转发，通过 Quantumult X、ShadowRocket 等代理软件连接中转机
  • Windows 平台使用 v2rayN v5.38 及后续版本
  • Android 平台使用 v2rayNG v1.7.27 及后续版本

注：实际所有平台都可使用中转机的方式实现，但需一层中转且消耗服务器流量，而且服务器通常是实名开通，只能作为过渡方案。

macOS

流量路径：浏览器 / 应用 –> 代理软件 –> Xray –> 代理服务器

安装 Xray

推荐使用 brew 安装，如果没有 brew 请先安装 brew：https://docs.brew.sh/Installation

使用 brew 安装 Xray

按照提示找到 Xray 的配置文件路径，通常 Apple Silicon 平台为 /opt/homebrew/etc/xray/config.json ，Intel 平台为 /usr/local/etc/xray/config.json，编辑此文件，内容如下，修改注释处配置

使用 brew services 管理服务

配置代理软件

v2rayU

协议选择 socks，端口填入上方配置文件里的「本地 socks 监听端口」

ClashX / ClashX Pro

配置文件格式参考：https://github.com/Dreamacro/clash/wiki/configuration

自动生成基础配置文件：https://sub.789.st/

在基础配置文件里添加本地 socks 服务器，端口为上方配置文件里的「本地 socks 监听端口」。规则部分保证 PROCESS-NAME,xray,DIRECT 置顶，避免 clash –> xray –> clash 导致回环

Chrome 配合 SwitchyOmega

参见：Chrome 配合 SwitchyOmega

端口为上方配置文件里的「本地 socks 监听端口」

Windows

流量路径：浏览器 / 应用 –> v2rayN –> 代理服务器

v2rayN

下载 v2rayN-Core.zip，如 Github 下载不动，可使用下面代理地址下载

Github 下载：https://github.com/2dust/v2rayN/releases/latest/download/v2rayN-Core.zip
代理地址下载：https://ghproxy.com/https://github.com/2dust/v2rayN/releases/latest/download/v2rayN-Core.zip

参数设置里选中「检查 Pre-release 更新」后，更新 Xray-Core，确保 Xray 版本在 1.7.3 及以上

如果下载 Xray-Core，可使用下面代理地址下载，解压后放入 v2rayN 文件夹内
代理地址下载：https://ghproxy.com/https://github.com/XTLS/Xray-core/releases/latest/download/Xray-windows-64.zip

添加 VLESS 服务器，设置如图所示，填入服务器地址、端口、ID 等

需确保「Core 类型」为 Xray，「流控」为 xtls-rprx-vision，「传输层安全」为 tls，「fingerprint」为 chrome

系统代理选择「自动配置系统代理」、路由选择「绕过大陆」

Android

v2rayNG

流量路径：浏览器 / 应用 –> v2rayNG –> 代理服务器

项目地址：https://github.com/2dust/v2rayNG/releases

手头没有安卓机，可参照 v2rayN 相关设置

Clash for Android

流量路径：应用 –> Clash for Android –> (代理软件支持的协议) –> 中转服务器 –> (VLESS + xtls-rprx-vision) –> 代理服务器
